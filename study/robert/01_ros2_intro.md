# 1장 ROS 2 소개

## 1.1 ROS 2란 무엇인가

ROS 2(Robot Operating System 2)는 로봇 소프트웨어를 개발하기 위한 오픈소스 프레임워크이다.  
이름에 Operating System이 들어가 있지만, 일반적인 운영체제 자체를 뜻하는 것은 아니다.  
ROS 2는 로봇을 개발할 때 자주 필요한 기능과 구조를 제공하는 **미들웨어 기반 소프트웨어 플랫폼**에 가깝다.

로봇을 개발하려면 센서 데이터를 읽고, 모터를 제어하고, 여러 소프트웨어 모듈이 서로 통신하며, 실시간에 가까운 반응을 해야 한다.  
이때 개발자는 모든 기능을 처음부터 직접 구현하기보다, 이미 잘 정리된 구조 위에서 필요한 기능을 조합해 빠르게 개발하는 것이 훨씬 효율적이다.  
ROS 2는 바로 이러한 문제를 해결하기 위해 사용된다.

---

## 1.2 ROS의 등장 배경

기존의 로봇 개발은 하드웨어마다 제어 방식이 다르고, 센서 종류도 다양하며, 소프트웨어 구조도 제각각이어서 재사용이 어려웠다.  
연구실이나 기업이 각각 독립적으로 개발하다 보니, 같은 기능을 반복해서 구현하는 일이 많았다.

이 문제를 해결하기 위해 등장한 것이 ROS이다.  
ROS는 로봇 개발자들이 공통적으로 사용하는 메시지 통신 방식, 패키지 구조, 개발 도구를 제공함으로써 로봇 소프트웨어의 재사용성과 확장성을 크게 높였다.

하지만 기존 ROS 1은 다음과 같은 한계를 가지고 있었다.

- 멀티 로봇 환경 대응이 제한적이었다.
- 실시간성 보장이 어려웠다.
- 보안 기능이 충분하지 않았다.
- 산업 현장에서 요구하는 안정성과 네트워크 유연성이 부족했다.
- 중앙 집중형 마스터 구조로 인해 확장성에 제약이 있었다.

이러한 한계를 개선하기 위해 새롭게 설계된 것이 ROS 2이다.

---

## 1.3 ROS 2의 핵심 특징

ROS 2는 단순히 ROS 1의 업그레이드 버전이 아니라, 현대적인 로봇 개발 환경에 맞춰 구조적으로 다시 설계된 플랫폼이다.

### 1.3.1 DDS 기반 통신

ROS 2는 DDS(Data Distribution Service)를 기반으로 동작한다.  
DDS는 산업용 분산 시스템에서 널리 사용되는 통신 기술로, 노드 간 데이터 송수신을 안정적으로 수행할 수 있도록 돕는다.

이를 통해 ROS 2는 다음과 같은 장점을 가진다.

- 중앙 마스터 없이 통신 가능
- 네트워크 환경에 유연하게 대응
- QoS(Quality of Service) 설정 가능
- 멀티 로봇 시스템에 적합
- 산업용 시스템과의 연계가 쉬움

### 1.3.2 노드 기반 구조

ROS 2에서는 기능 단위를 **노드(Node)** 라고 부른다.  
예를 들어 카메라를 읽는 노드, 라이다를 처리하는 노드, 모터를 제어하는 노드가 각각 독립적으로 존재할 수 있다.

이 구조의 장점은 다음과 같다.

- 기능별 분리로 유지보수가 쉬움
- 특정 기능만 교체하거나 수정 가능
- 협업 개발에 유리
- 시스템 확장성이 높음

### 1.3.3 메시지 중심 통신

ROS 2의 각 노드는 서로 메시지를 주고받으며 동작한다.  
대표적인 통신 방식은 다음과 같다.

- Topic: 비동기 데이터 스트림 전달
- Service: 요청과 응답 기반 통신
- Action: 시간이 걸리는 작업을 위한 통신 방식

이러한 구조 덕분에 로봇 시스템을 느슨하게 결합된 형태로 설계할 수 있다.

### 1.3.4 다양한 언어 지원

ROS 2는 주로 다음 언어를 사용한다.

- C++
- Python

C++은 성능이 중요한 제어 로직에 많이 사용되고, Python은 빠른 프로토타이핑과 학습용 예제 제작에 적합하다.

---

## 1.4 ROS 2가 사용되는 분야

ROS 2는 단순한 학습 도구가 아니라 실제 산업 현장과 연구 환경에서 널리 사용된다.

### 대표 활용 분야

- 자율주행 로봇
- 물류 로봇
- 서비스 로봇
- 로봇 팔 및 산업용 자동화 장비
- 드론 및 무인 이동체
- 스마트 팩토리
- 연구용 로봇 플랫폼
- 교육용 로봇 시스템

특히 여러 센서와 제어 모듈을 통합해야 하는 복잡한 로봇 개발에서 ROS 2의 장점이 크게 드러난다.

---

## 1.5 ROS 1과 ROS 2의 차이

ROS 1과 ROS 2는 이름은 비슷하지만 내부 구조와 철학에서 차이가 크다.

| 구분 | ROS 1 | ROS 2 |
|------|------|------|
| 통신 구조 | ROS Master 기반 | DDS 기반 분산 통신 |
| 실시간성 | 제한적 | 개선됨 |
| 보안 | 기본 지원 부족 | 보안 구조 확장 가능 |
| 멀티 로봇 | 설정이 복잡함 | 비교적 유리 |
| 산업 적용 | 연구 중심 | 산업 적용 확대 |
| 지원 OS | 주로 Linux | Linux, Windows, macOS 등 |

ROS 2는 현대적인 로봇 개발 환경에 더 적합한 방향으로 발전한 플랫폼이라고 볼 수 있다.

---

## 1.6 ROS 2를 배우는 이유

ROS 2를 배우면 다음과 같은 이점이 있다.

### 1) 로봇 개발의 표준 구조를 익힐 수 있다
ROS 2는 전 세계적으로 널리 사용되는 로봇 개발 방식 중 하나이다.  
이를 익히면 로봇 시스템을 체계적으로 설계하는 방법을 자연스럽게 배울 수 있다.

### 2) 재사용 가능한 개발 습관을 만들 수 있다
패키지, 노드, 메시지, 서비스 등의 개념을 활용하면 유지보수가 쉬운 구조로 개발할 수 있다.

### 3) 시뮬레이션과 실제 로봇을 연결하기 쉽다
ROS 2는 Gazebo, RViz, MoveIt 등 다양한 도구와 연동이 가능해 가상 환경과 실제 하드웨어를 함께 다루기 좋다.

### 4) 산업과 연구를 동시에 준비할 수 있다
ROS 2는 교육용 예제부터 실제 서비스 로봇 개발까지 폭넓게 활용되므로 장기적으로 매우 유용한 역량이 된다.

---

## 1.7 정리

ROS 2는 로봇 개발을 위한 현대적인 오픈소스 프레임워크이다.  
노드 기반 구조, 메시지 중심 통신, DDS 기반 분산 설계, 다양한 운영체제 지원 등의 특징을 통해 복잡한 로봇 시스템을 효율적으로 개발할 수 있게 해준다.

앞으로 ROS 2를 학습한다는 것은 단순히 도구 하나를 익히는 것이 아니라,  
**복잡한 로봇 시스템을 구조적으로 설계하고 구현하는 방법**을 배우는 과정이라고 할 수 있다.
