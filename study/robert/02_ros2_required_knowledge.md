# 2장 ROS 2 기반 로봇 개발에 필요한 정보

## 2.1 왜 사전 지식이 중요한가

ROS 2는 매우 강력한 로봇 개발 프레임워크이지만, 아무 배경지식 없이 접근하면 구조가 복잡하게 느껴질 수 있다.  
특히 ROS 2는 단순한 프로그래밍 도구가 아니라 운영체제, 네트워크, 센서, 제어, 소프트웨어 구조 등 여러 요소가 함께 작동하는 환경이다.

따라서 ROS 2를 제대로 이해하고 활용하기 위해서는 몇 가지 필수 개념을 먼저 알고 있어야 한다.  
이 장에서는 ROS 2 개발을 시작하기 전에 알아두면 좋은 핵심 정보를 정리한다.

---

## 2.2 기본적으로 필요한 기술 영역

ROS 2 기반 로봇 개발에는 다음과 같은 기술 영역이 함께 사용된다.

- Linux 사용 능력
- Python 또는 C++ 프로그래밍
- 터미널 명령어 이해
- 패키지 관리 개념
- 네트워크와 프로세스 개념
- 센서와 액추에이터 이해
- 좌표계와 메시지 구조 이해

모든 내용을 처음부터 완벽히 알아야 하는 것은 아니지만, 최소한 기본 개념은 알고 시작하는 것이 좋다.

---

## 2.3 Linux 환경 이해

ROS 2는 다양한 운영체제를 지원하지만, 실제 개발에서는 Linux, 특히 Ubuntu 환경이 가장 널리 사용된다.  
따라서 ROS 2를 다루려면 Linux 기본 사용법을 익히는 것이 중요하다.

### 알아두어야 할 Linux 기본 요소

- 디렉터리 구조 이해
- 파일 생성, 복사, 이동, 삭제
- 권한 개념
- 환경 변수
- 패키지 설치와 업데이트
- 쉘(Shell) 사용법

### 자주 사용하는 명령어 예시

- `pwd`: 현재 위치 확인
- `ls`: 파일 목록 확인
- `cd`: 디렉터리 이동
- `mkdir`: 폴더 생성
- `rm`: 파일 삭제
- `cp`: 파일 복사
- `mv`: 파일 이동
- `chmod`: 권한 변경
- `echo`: 문자열 출력
- `source`: 환경설정 반영

ROS 2는 터미널 기반 작업이 많기 때문에, GUI보다 명령행 환경에 익숙해지는 것이 중요하다.

---

## 2.4 Python과 C++ 기초

ROS 2 개발에서는 주로 Python과 C++이 사용된다.

### Python이 적합한 경우

- 빠르게 예제를 만들고 싶을 때
- 학습용 코드 작성
- 테스트 자동화
- 간단한 제어 로직 구현

### C++이 적합한 경우

- 고성능 처리
- 실시간성에 가까운 제어
- 대규모 시스템 최적화
- 하드웨어 제어와 밀접한 로직

초보자라면 Python으로 시작한 뒤, 성능 최적화가 필요한 부분에서 C++로 확장하는 방식이 일반적이다.

### 최소한 필요한 프로그래밍 개념

- 변수와 자료형
- 조건문과 반복문
- 함수
- 클래스와 객체
- 모듈 import
- 예외 처리
- 파일 입출력
- 라이브러리 사용법

ROS 2 자체를 배우는 것과 언어 문법을 배우는 것이 동시에 진행되면 부담이 크므로, 기본 문법은 먼저 익혀두는 것이 좋다.

---

## 2.5 패키지와 의존성 개념

ROS 2에서 프로젝트의 기본 단위는 **패키지(Package)** 이다.  
패키지는 실행 코드, 설정 파일, 메시지 정의, 런치 파일 등을 하나의 단위로 묶는 구조이다.

### 패키지를 이해해야 하는 이유

- 기능별로 프로젝트를 나눌 수 있음
- 재사용과 배포가 쉬움
- 의존성 관리가 가능함
- 협업 시 구조가 명확해짐

### 의존성이란 무엇인가

어떤 패키지가 다른 패키지나 라이브러리에 의존하는 관계를 말한다.  
예를 들어 센서 데이터를 처리하는 패키지가 `rclpy`, `sensor_msgs` 같은 라이브러리를 필요로 할 수 있다.

의존성 관리는 프로젝트 안정성과 직결되므로 매우 중요하다.

## 2.6 노드, 토픽, 서비스, 액션 이해

ROS 2를 배우기 위해 가장 먼저 익혀야 할 핵심 개념은 다음 네 가지이다.

### 2.6.1 노드(Node)

노드는 하나의 기능을 수행하는 독립 실행 단위이다.  
예를 들어 다음과 같은 노드가 있을 수 있다.

- 카메라 입력 노드
- 장애물 인식 노드
- 속도 제어 노드
- 로봇 상태 표시 노드

### 2.6.2 토픽(Topic)

토픽은 노드 간 데이터를 비동기적으로 주고받는 방식이다.  
한 노드가 데이터를 발행(Publish)하면, 다른 노드가 이를 구독(Subscribe)한다.

예시:
- 카메라 노드가 이미지 데이터를 발행
- 인식 노드가 이미지 데이터를 구독

### 2.6.3 서비스(Service)

서비스는 요청(Request)과 응답(Response) 방식의 통신이다.  
특정 작업을 즉시 요청하고 그 결과를 받는 데 적합하다.

예시:
- 로봇의 현재 좌표를 요청하고 응답 받기
- 센서 초기화 요청 보내기

### 2.6.4 액션(Action)

액션은 완료까지 시간이 걸리는 작업을 처리할 때 사용된다.  
중간 진행 상태를 확인하거나 취소할 수도 있다.

예시:
- 목표 지점까지 이동
- 로봇 팔로 물체 집기
- 경로 탐색 수행

---

## 2.7 메시지 타입 이해

ROS 2에서 노드는 메시지를 통해 통신하므로 메시지 구조를 이해하는 것이 중요하다.

### 대표 메시지 예시

- `std_msgs/msg/String`
- `geometry_msgs/msg/Twist`
- `sensor_msgs/msg/Image`
- `sensor_msgs/msg/LaserScan`
- `nav_msgs/msg/Odometry`

각 메시지는 특정한 데이터 구조를 가지고 있으며, 목적에 맞게 사용해야 한다.

예를 들어 `Twist` 메시지는 선속도와 각속도를 표현하므로 이동 로봇 제어에 자주 사용된다.

---

## 2.8 좌표계와 TF 개념

로봇 개발에서는 위치와 방향을 정확하게 표현하는 것이 매우 중요하다.  
이를 위해 좌표계(frame) 개념과 TF(Transform)를 이해해야 한다.

### 자주 등장하는 좌표계

- `map`: 전체 지도 기준 좌표계
- `odom`: 누적 이동 기반 좌표계
- `base_link`: 로봇 본체 기준 좌표계
- `laser`, `camera_link`: 센서 기준 좌표계

TF는 서로 다른 좌표계 사이의 관계를 계산하고 관리하는 기능을 제공한다.  
이 개념을 이해하지 못하면 센서 데이터와 로봇 자세를 연결하기 어렵다.

---

## 2.9 센서와 액추에이터 이해

ROS 2 기반 로봇 개발은 결국 실제 하드웨어와 연결된다.  
따라서 주요 센서와 액추에이터의 역할을 이해하는 것이 중요하다.

### 대표 센서

- 카메라
- 라이다(LiDAR)
- 초음파 센서
- IMU
- 엔코더
- GPS

### 대표 액추에이터

- DC 모터
- 서보 모터
- 스테퍼 모터
- 로봇 팔 구동 장치

센서는 입력을 담당하고, 액추에이터는 출력을 담당한다.  
ROS 2는 이 둘 사이의 데이터를 연결하고 제어 로직을 구성하는 중심 역할을 한다.

---

## 2.10 빌드 시스템 이해

ROS 2는 패키지를 빌드할 때 `colcon`을 사용한다.  
여러 패키지를 함께 빌드하고 의존성을 고려해 전체 워크스페이스를 관리할 수 있다.

### 기본 개념

- Workspace: 여러 패키지를 모아 개발하는 작업 공간
- `src`: 패키지 소스 코드 저장 디렉터리
- build: 빌드 중 생성되는 파일
- install: 실행에 필요한 설치 결과물
- log: 빌드 로그

빌드 시스템을 이해하면 패키지 구조와 실행 흐름을 더 명확히 파악할 수 있다.

---

## 2.11 디버깅과 로그 확인 습관

로봇 개발은 하드웨어와 소프트웨어가 함께 동작하므로 오류 원인이 다양하다.  
따라서 다음과 같은 기본 디버깅 습관이 중요하다.

- 터미널 로그를 꼼꼼히 읽기
- 노드 실행 여부 확인
- 토픽 발행/구독 상태 확인
- 메시지 타입 확인
- 환경 변수 설정 상태 확인
- 빌드 에러와 런타임 에러 구분하기

ROS 2에서는 다양한 CLI 명령어를 통해 시스템 상태를 점검할 수 있다.

예시:
- `ros2 node list`
- `ros2 topic list`
- `ros2 topic echo`
- `ros2 service list`
- `ros2 action list`

## 2.12 정리

ROS 2 기반 로봇 개발을 위해서는 단순히 명령어만 외우는 것이 아니라,  
Linux, 프로그래밍, 패키지 구조, 메시지 통신, 좌표계, 센서와 제어 등 여러 기초 개념을 함께 이해해야 한다.

이러한 사전 지식은 처음에는 많아 보일 수 있지만, 실제로는 서로 유기적으로 연결되어 있다.  
기초를 탄탄히 다져두면 이후의 ROS 2 학습 속도와 이해도가 크게 높아진다.
