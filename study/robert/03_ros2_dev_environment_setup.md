# 3장 ROS 2 개발환경 구축

## 3.1 개발환경 구축의 중요성

ROS 2를 제대로 학습하고 개발하기 위해서는 안정적인 개발환경이 필요하다.  
개발환경이 올바르게 구성되지 않으면 설치 오류, 패키지 충돌, 실행 실패, 통신 문제 등이 자주 발생한다.  
따라서 첫 단계에서 환경을 정확히 설정하는 것은 매우 중요하다.

ROS 2 개발환경은 단순히 프로그램 하나를 설치하는 수준이 아니라 다음 요소들이 함께 맞물려야 한다.

- 운영체제
- ROS 2 배포판
- Python 및 C++ 개발 도구
- 빌드 도구
- 워크스페이스 구조
- 환경 변수
- 시뮬레이터 및 시각화 도구

이 장에서는 초보자도 이해할 수 있도록 ROS 2 개발환경 구축 흐름을 정리한다.

---

## 3.2 권장 운영체제

ROS 2는 Linux, Windows, macOS를 지원하지만, 가장 일반적이고 안정적인 환경은 Ubuntu Linux이다.

### Ubuntu가 많이 사용되는 이유

- 공식 문서와 예제가 풍부함
- 패키지 설치가 비교적 쉬움
- 커뮤니티 정보가 많음
- 대부분의 로봇 개발 사례가 Ubuntu 기준으로 설명됨

학습과 실습을 목적으로 한다면 Ubuntu 환경을 우선적으로 추천한다.

---

## 3.3 설치 전 준비사항

ROS 2를 설치하기 전에 다음을 확인하는 것이 좋다.

### 하드웨어 권장 조건

- 64비트 CPU
- 메모리 8GB 이상 권장
- 여유 저장공간 20GB 이상 권장
- 인터넷 연결 가능 환경

### 기본 소프트웨어 준비

- 터미널 사용 가능
- 시스템 업데이트 완료
- Python 설치 확인
- Git 설치
- 개발 도구 설치 가능 여부 확인

개발용 노트북이나 데스크톱에서 직접 설치하는 방법이 가장 일반적이지만, 상황에 따라 가상머신이나 Docker를 사용할 수도 있다.

---

## 3.4 ROS 2 설치 방식 이해

ROS 2는 여러 방식으로 설치할 수 있다.

### 1) 패키지 매니저를 통한 설치
운영체제의 패키지 관리 시스템을 이용해 설치하는 방식이다.  
가장 간편하고 초보자에게 적합하다.

### 2) 소스 빌드 설치
ROS 2 소스코드를 직접 받아 빌드하는 방식이다.  
최신 기능 사용이나 커스터마이징이 필요할 때 유용하지만 난이도가 높다.

### 3) Docker 기반 설치
컨테이너 환경에서 ROS 2를 실행하는 방법이다.  
환경 격리와 배포에 유리하지만, 입문자에게는 다소 복잡할 수 있다.

처음 학습할 때는 일반적으로 **패키지 매니저 기반 설치**가 가장 적합하다.

---

## 3.5 개발에 필요한 기본 도구

ROS 2 설치 후에는 다음 도구들을 함께 준비해야 한다.

### 필수 도구

- Git: 소스코드 관리
- Python3: 스크립트 및 노드 개발
- pip: Python 패키지 관리
- colcon: ROS 2 워크스페이스 빌드
- vcstool: 여러 저장소 관리
- rosdep: 의존성 자동 설치

### 자주 사용하는 부가 도구

- VS Code: 코드 편집기
- tmux: 터미널 세션 관리
- RViz2: 데이터 시각화
- Gazebo: 시뮬레이션
- rqt: GUI 기반 분석 도구

---

## 3.6 환경 변수 설정

ROS 2를 설치한 뒤 가장 중요한 작업 중 하나는 환경 변수를 올바르게 적용하는 것이다.  
환경 변수가 적용되지 않으면 `ros2` 명령어가 인식되지 않거나, 패키지를 찾지 못하는 문제가 생긴다.

대표적으로 다음과 같은 설정이 필요하다.

- ROS 2 설치 환경 로드
- 워크스페이스 install 경로 로드
- 쉘 시작 시 자동 반영 설정

보통 터미널에서 `source` 명령어를 사용해 현재 세션에 설정을 반영한다.

예를 들어 ROS 2 설치 경로와 워크스페이스 환경을 각각 로드해야 한다.  
이 과정을 자동화하기 위해 쉘 설정 파일에 등록하기도 한다.

---

## 3.7 워크스페이스 구조 이해

ROS 2 개발은 보통 워크스페이스 단위로 진행한다.  
워크스페이스는 여러 패키지를 함께 관리하는 개발 공간이다.

### 일반적인 워크스페이스 구조

- `workspace/`
  - `src/`
  - `build/`
  - `install/`
  - `log/`

### 각 폴더의 역할

- `src`: 소스코드와 패키지를 저장하는 위치
- `build`: 빌드 중 생성되는 중간 파일
- `install`: 실행 가능한 결과물 저장 위치
- `log`: 빌드와 실행 관련 로그 저장

초보자는 `src` 폴더에 패키지를 추가하고, `colcon build`를 통해 전체 워크스페이스를 빌드하는 흐름을 먼저 익히면 된다.

## 3.8 첫 패키지 생성 흐름

개발환경이 준비되면 보통 첫 번째 ROS 2 패키지를 생성하며 실습을 시작한다.

### 기본 흐름

1. 워크스페이스 생성
2. `src` 디렉터리 생성
3. 패키지 생성 명령 실행
4. 빌드 수행
5. 환경 변수 적용
6. 노드 실행 확인

이 과정을 통해 ROS 2의 기본 개발 사이클을 익힐 수 있다.

### 개발 사이클 예시

- 코드 작성
- 빌드
- source 적용
- 실행
- 로그 확인
- 수정 후 재빌드

이 반복 구조가 ROS 2 개발의 핵심 습관이 된다.

## 3.9 에디터와 개발 생산성 도구

ROS 2는 터미널 기반 개발이 많지만, 생산성을 높이기 위해 좋은 에디터를 사용하는 것이 좋다.

### 추천 기능

- Python/C++ 문법 강조
- 자동완성
- Git 연동
- 터미널 내장
- 디버깅 지원
- 확장 프로그램 지원

VS Code는 비교적 가볍고 설정이 쉬워 많은 개발자들이 사용한다.  
C++ 확장, Python 확장, ROS 관련 확장을 활용하면 개발이 편리해진다.

---

## 3.10 시뮬레이션 환경 준비

실제 로봇 하드웨어가 없어도 ROS 2를 학습할 수 있도록 다양한 시뮬레이션 도구가 제공된다.

### 대표 시뮬레이션/시각화 도구

- Gazebo: 물리 기반 로봇 시뮬레이션
- RViz2: 센서, 맵, 좌표계, 경로 시각화
- rqt: 그래프 및 상태 확인 GUI 도구

### 시뮬레이션 환경이 중요한 이유

- 실제 하드웨어 없이도 테스트 가능
- 위험한 동작을 안전하게 검증 가능
- 알고리즘 개발 속도 향상
- 센서와 제어 구조를 반복적으로 실험 가능

초보자라면 실제 로봇을 연결하기 전에 시뮬레이터와 RViz를 먼저 익히는 것이 좋다.

## 3.11 자주 발생하는 초기 문제

ROS 2 개발환경을 구축할 때 초보자가 자주 겪는 문제는 다음과 같다.

### 1) `ros2` 명령어가 동작하지 않음
주로 환경 변수가 제대로 적용되지 않았을 때 발생한다.

### 2) 빌드는 되었지만 실행이 안 됨
패키지 의존성, 실행 파일 등록, source 누락 등의 원인이 많다.

### 3) 패키지를 찾지 못함
워크스페이스를 빌드한 뒤 install 환경을 다시 로드하지 않아 발생할 수 있다.

### 4) Python 모듈 에러 발생
패키지 의존성 누락이나 Python 환경 충돌일 수 있다.

### 5) 토픽 통신이 보이지 않음
노드 실행 여부, 도메인 설정, QoS 설정 차이 등을 확인해야 한다.

이러한 문제는 대부분 환경 설정과 빌드 흐름을 정확히 이해하면 해결 가능하다.

## 3.12 추천 학습 순서

개발환경을 구축한 뒤에는 다음과 같은 순서로 학습하면 효율적이다.

1. `ros2` CLI 명령어 익히기
2. 노드 실행해보기
3. 토픽 발행과 구독 실습
4. 서비스와 액션 이해
5. 패키지 직접 생성해보기
6. RViz2로 데이터 시각화
7. Gazebo 시뮬레이션 적용
8. 실제 센서 또는 로봇 연결

이 순서를 따르면 추상적인 개념이 점점 실제 동작과 연결되기 시작한다.

## 3.13 정리

ROS 2 개발환경 구축은 단순한 설치 작업이 아니라 이후의 학습과 프로젝트 품질을 결정하는 중요한 출발점이다.  
운영체제 선택, 필수 도구 설치, 환경 변수 설정, 워크스페이스 구성, 시뮬레이션 도구 준비까지 차근차근 진행해야 안정적인 개발 기반을 만들 수 있다.

환경이 잘 갖춰지면 ROS 2는 훨씬 친숙한 개발 도구가 된다.  
다음 장에서는 많은 개발자와 기업이 왜 ROS 2로 이동하고 있는지, 그리고 왜 지금 ROS 2를 선택해야 하는지를 살펴본다.
